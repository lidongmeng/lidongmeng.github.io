链表是否存在环问题

判断链表是否有环及其证明

利用快慢指针，快指针每次走两步，慢指针每次走一步，如果相遇则有环。

算法证明

如下图：

从开始节点到环的开始节点之间的距离是m，设环的长度是n，当慢指针走到环的开始位置时候，快指针这时候走到环中m%n的位置，假设他们能够相遇，并且假设是在慢指针进入环t步之后相遇的，呢么此时:
慢指针的位置：m + t%n
快指针的位置：m + (m + 2*t)%n
相遇表明m + t%n = m + (m + 2*t)%n，所以可以得到(m+t)%n = 0;
由于m是正整数，n也是正整数，而且是固定的，所以肯定存在一个t使得上式成立并且：t = n*p - m(p是走过的圈数).

算法实现

有环链表开始节点查找

算法证明

有上述证明可以看出相遇时候(m+t)%n=0,假设相遇时候慢指针距离环开始节点距离为X，则X=t%n(t=X+n*p,p表示走过的圈数),假设剩余环的长度为Y，如下图所示:

从上面分析可以得出下面三个式子:

  1. (m + t) % n = 0;
  2. t = X + n*p;
  3. n = X + Y;
    所以，可以看出：
    (m+t)%n = (m+X+n*P)%n = (m+n-Y+n*p)%n = (m-Y)%n + (p+1)*n%n = 0;
    也就是：m = Y + n*q;
    所以我们只需要将一个指针移动到链表的头节点处，然后依次向下一个节点移动，再次相遇的位置就是环的开始节点位置。

最长等差子序列长度

发表于 2016-04-24   |   分类于 算法   |  

题目描述

一个无序数组，求它的最长的等差子序列的长度。

题目分析

首先题目并没有说得到的等差子序列要和原数组中的位置是否一致，所以要分两种情况来进行分析。

等差子序列要和原数组元素位置一致

我们可以构造一个map，key是等差数列的d,而value是两个index，开始位置i和结束位置j，并且保证i小于j，然后对这个map进行扫描，对每一个d，先对后面的数组按照开始位置i进行排序，然后顺序查找，下一个的j等于上一个的i表示能够连接起来，

等差子序列不一定和原数组元素位置一致

拓展

leetcode有一道题目求最长连续子序列的长度。(1，2，3叫做连续，1，2，4不叫连续)
首先，你当然可以先排序，然后去遍历，但是时间复杂度O(nlgn);

是否可以用O(n)的时间复杂度来做？这时候就要用额外的空间来存储当前数字能够到达的边界位置。

二叉树遍历之前序遍历

发表于 2016-04-20   |   分类于 数据结构与算法   |  

面试中二叉树的各种遍历方法经常被问到，最先想到的可能就是递归遍历，但是面试官是不会满足于此的，一般会让你写出来非递归的遍历方法，最近看到Morris遍历方法，甚是巧妙，竟然不用额外空间就可以得到遍历结果，所以写此文来记录~

二叉树基本数据结构定义

递归遍历

前序遍历是先访问根节点，并输出，然后递归遍历左子树，递归遍历右子树，递归终止条件是当前遍历到的节点为空。

非递归使用额外空间栈遍历

递归实际是靠系统自身的递归栈来记录信息，我们可以直接用栈来模拟递归过程，用一个栈记录访问过的节点，用一个额外指针记录当前访问到的节点，访问节点如果不为空，打印当前节点，并向左子树方向前进，直到当前节点为空，这时候已经走到左子树的尽头，然后从栈中恢复到父节点，并像右节点前进，重复上述过程，一直到栈空而且当前节点空为止。

Morris遍历

如果不能利用额外栈空间，如何做到遍历？实际最大困难在于如何从叶子节点回到上一层节点，Morris利用了线索二叉树的方法，Morris方法中不需要为每个节点额外分配指针指向其前驱（predecessor）和后继节点（successor），只需要利用叶子节点中的左右空指针指向某种顺序遍历下的前驱节点或后继节点就可以了。

步骤

  1. 如果当前节点的左孩子为空，则输出当前节点并将其右孩子作为当前节点。
  2. 如果当前节点的左孩子不为空，在当前节点的左子树中找到当前节点在中序遍历下的前驱节点。
    a) 如果前驱节点的右孩子为空，将它的右孩子设置为当前节点。输出当前节点（在这里输出，这是与中序遍历唯一一点不同）。当前节点更新为当前节点的左孩子。
    b) 如果前驱节点的右孩子为当前节点，将它的右孩子重新设为空。当前节点更新为当前节点的右孩子。
  3. 重复以上1、2直到当前节点为空。

实现

复杂度

时间复杂度：O(n)
空间复杂度：O(1)
