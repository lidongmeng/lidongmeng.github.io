IPC之SystemV信号量

发表于 2016-06-13   |  

Posix信号量虽然有有名和无名之分，但是都只是针对一个资源的数量的操作，但是SystemV信号量却可以针对一个或者多个信号量(计数信号量集)，所以增加了操作复杂性。

systemv信号量相关操作函数

semget函数

成功返回信号量标识符，错误返回-1;函数功能是填写semid_ds中的相关数据sem_perm通过最后一个参数填写，sem_nsems通过nsems填写，sem_otime被置为0，sem_ctime被置为当前时间。但是semget函数并不初始化各个信号量的semval，要通过semctl来进行初始化操作
创建和初始化是两个原子性操作，在创建后未进行初始化就有可能被别的进程抢占，而且直接使用，但是此时的信号量的值是不确定的，大写的尴尬，不过还好的是有sem_otime这个选项保证了未初始化时候为0，所以在别的进程使用信号量时候，得到信号量标识符后，不要直接操作信号p的v操作，先使用semctl的IPC_STAT命令循环等待sem_otime变为非0，也就是阻塞在这里。

semop函数–pv操作主要函数

第二个参数表示的是对一个信号量的具体操作：

这个结构体不一定只有这三个数据项，但至少要初始化这三个项目，内核保证传递给semop函数的这个操作数组被原子地操作。不过这三个参数还是要好好理解下：
1、 sem_num是对应值0-nsems-1，是处理哪一个信号量；
2、 sem_op是正数时候，表示v操作，释放资源，将对应信号量加上1;如果是0，表示调用者希望等待该信号量的值变为0，对应着信号量sem中的semzcnt这个变量，如果semval为0立即返回，不为0阻塞并使semzcnt增加1;如果是负数，则表示调用者希望等待semval变为大于或者等于sem_op的绝对值（可以请求一个或者多个资源），对应于p操作，如果semval的值小于sem_op绝对值，就减去sem_op绝对值然后返回，如果不大于，就阻塞等待。
3、 第三个sem_flag可以为0，IPC_NOWAIT,SEM_UNDO三个，如果为0，表示正常操作，如果为IPC_WAIT，使对信号量的操作时非阻塞的。即指定了该标志，调用线程在信号量的值不满足条件的情况下不会被阻塞，而是直接返回-1，并将errno设置为EAGAIN。如果为SEM_UNDO，那么将维护进程对信号量的调整值，以便进程结束时恢复信号量的状态。

sem_ctl函数—控制操作

对信号量的各种控制操作，第二个参数表示所要操作的信号量的下标，只有在第三个是GETVAL, SETVAL, GETNCNT, GETZCNT以及GETPID时候才会用。第三项是具体的操作，第四项是一个联合体，这个联合体系统不提供使用时候要自己申明：

这个联合体，第一个是设置单个信号量值时候来设置它的值(SETVAL),第二个在IPC_SET和IPC_STAT使用；第三个是一个数组，是获取或者设置所有信号量值时候传入数组(SETALL,GETALL)。
第三个参数有很多选项：
1、 GETVAL把对应下标为semnum的sem的semval返回;SETVAL和GETVAL对应，设置semval值为arg.val值，如果成功，相应信号量在所有进程中的信号量调整值为0.
2、 GETPID,GETNCNT,GETZCNT都不用第四个参数；
3、 GETALL返回所有信号量的semval，并放入值到arg.array中；SETALL设置所有信号量的值为array数组指定的值。
4、 IPC_RMID是删除这个信号量集； (对于一个持有该信号量的进程在没有释放该信号量的情况下就终止了。那么该进程所占用的信号量内核是不会进行释放的。会立即将该信号量从系统中彻底删除，不能再对该信号量进行任何访问。)
5、 IPC_SET是设置信号量集的sem_perm.uid,sem_perm.gid和sem_perm.mode三个参数，由arg的第二项参数相应值来设置。
6、 IPC_STAT返回指定信号量集中semid_ds结构，通过arg.buf返回，调用者首先要分配一个semid_ds结构并把arg.buf指向这个结构。

IPC之Posix消息队列

发表于 2016-06-11   |   分类于 IPC   |  

Posix消息队列是Linux IPC的一种通信方式，通常用于在不同进程之间发送特定格式的消息数据。有足够写权限的线程可以往队列中放置消息，有足够读权限的线程可以从队列中读取消息。而且消息队列具有随内核的持续性，即使进程关闭了消息队列，只要引用计数大于0，此消息队列依然存在。

Posix消息队列基本函数

mq_open

创建一个消息队列，name为消息队列的名字，oflag和open函数类似，选项可以为O_RDONLY,O_WRONLY,O_RDWR,O_NONBLOCK,O_CREAT,O_EXCL等，mode是可选的，只有oflag中有O_CREAT并且消息队列不存在时候才需要，attr是属性设置，只能设置最大消息数目和消息的最大的size。返回消息队列的描述符。

mq_close

关闭一个打开的消息队列，但是并不从系统中删除，一个进程结束会自动关闭打开的消息队列。

mq_unlink

删除一个消息队列，消息队列创建后只能通过该函数或者是内核自举才能进行删除，每个消息队列有一个引用计数，只有计数为0才真正从系统中删除。

mq_getattr/mq_setattr

消息队列中每个消息具有三个属性：一个优先级别的整数，消息部分长度，消息数据本身，消息队列的可能布局如下图所示：

而消息队列的队列属性至少要有四个：消息队列标志(是否阻塞)，最大消息数目，最大消息的字节数，现有消息的数目。

消息队列属性设置和获取可以通过：

注意：mq_setattr可以设置的属性只有mq_flags，用来设置或者清除消息队列的非阻塞标志，其他属性忽略，mq_maxmsg和mq_msgsize只能在mq_open时候设置，并且mq_open只设置这两个属性，其余的忽略。

mq_send

向消息队列中写入一条消息，第二个参数是一个指向消息体缓冲区的指针，消息长度，以及消息的优先级别，数值越大，级别越高。

mq_receive

取走一条消息，msg_len不能小于消息队列中消息的最大大小，而且返回的是消息队列中优先级别最高的消息。一般发送和接受是阻塞的，但是可以设置非阻塞。

mq_notify

Posix消息队列可以通过异步通知方式通知消息到来，通知方式可以有两种，SIGEV_SIGNAL以信号方式通知，SIGEV_THREAD以线程方式通知。
信号方式通知，要设置：sigev_signo和sigev_value；
线程方式通知，要设置：sigev_notify_function和sigev_notify_attributes

注意：只是在消息队列从空到非空时才产生消息通知事件, 而且这种注册方式是一次性的

信号通知方式

信号方式通知，要设置：sigev_signo和sigev_value；而且这种通知只有在一个消息到达所指定的先前为空的队列时候才会得到通知。

全局变量代替信号处理函数中读取消息

想不断接受消息，则必须在接受消息之前(mq_receive之前)再次注册信号notify，不能在之后因为mq_notify和mq_receive都不是异步安全函数，可能出现安全性问题，比方在处理时候被重入新的消息就尴尬了。可以通过设置全局变量，不在信号处理函数中读取消息，而在主线程中处理消息。

非阻塞方式读取队列

但是上述程序如果在能够读取第一个消息前有两个消息到达，呢么只有一个通知发出，读取第一个消息后，sigsuspend等待另一个消息，对应它的通知永远不会到来。所以当使用mq_notify产生信号时候，要以非阻塞模式读取消息队列。

使用sigwait函数处理特定的信号

这种非阻塞模式处理不会再有任何问题，但是效率可能不是太高，调用sigsuspend阻塞等待消息到达，有一个消息到来时候，信号产生，主线程被阻止转去执行信号处理函数，然后执行并设置mqflag标志，主线程再次运行，读出消息，但是更为高效的方法是阻塞在某个函数上面，仅仅等待该信号的提交，而不是让内核执行一个只为设置一个标志的信号处理程序，使用sigwait可以达到此种目的，有点类似select的作用。

使用方法：调用sigwait前先阻塞某个信号集，然后将这个信号集指定为set参数，一直阻塞等到一个或者多个信号待处理，然后返回一个信号，该信号值通过sig返回，这个过程可以称作为同步地等待一个异步事件。

select方式

消息队列描述符和普通描述符不同，不能使用在io多路复用函数中，但是可以使用管道和mq_notify函数配合来处理，因为write函数是异步信号安全的，所以可以在信号处理函数中调用write向管道写入一个字节来触发select。

遇到Interrupted system call错误，错误原因是收到signal信号select会返回错误并且errno为EINTR，这个时候不应该退出。

线程通知方式

线程方式通知，要设置：sigev_notify_function和sigev_notify_attributes

程序中出现了message too long错误，原因是mq_receive函数的第三个参数的值设置错误，要大于等于mq_msgsize。

Posix 消息队列和System V消息队列

posix消息队列与system v消息队列的差别：
1、对posix消息队列的读总是返回最高优先级的最早消息，对system v消息队列的读则可以返回任意指定优先级的消息；
2、 当往一个空队列放置一个消息时，posix消息队列允许产生一个信号或启动一个线程，system v消息队列则不提供类似机制。
3、 System V提供了从队列中取出特定优先级的消息。

Linux中有两种类型的IO，一种是ANSIC提供的高级IO,叫做带缓冲的IO，另外一种是低级IO，系统调用的IO，不带缓冲，带不带缓冲取决于调用一次是否是内核的一个系统调用。

文件描述符

对应Linux内核来说，每一个打开的文件通过文件描述符引用，读写文件都通过该文件描述符进行操作，系统默认的3个文件0,1,2分别代表标准输入，标准输出以及标准错误。一个进程能打开的最多描述符个数是1024个(ulimit -a/-n)，一个系统打开的最多文件数目查看(cat /proc/sys/fs/file-max)，如下图所示：

POD类型

C风格的struct叫做POD(Plain Old Data)对象，仅仅包含数据成员(基本类型，任何类型的指针或者引用)，任何POD对象的初始化都可以使用memset()函数，初始化memset(&s, 0, sizeof(Test)),定义时候也可以直接指定初值，`Test t = {0};第一个成员初始化为0，其余的自动初始化为0.

位域

从字节对齐可以学到，在设计类或者struct的时候，成员变量的成员的声明顺序要从大到小，因为对象头部不可能填充字节如果从小到大就会在中间填充空洞，而从大到小声明则只会在末尾处填充。

Union

联合提供了一种使不同类型数据成员之间共享存储空间的方法，同时可以实现不同类型数据成员之间的自动类型转换，但是同一时间只能存储一个成员的值，占用的内存大小取决于字节数最多的那个变量。C++中的Union还可以定义成员的访问说明符，定义成员函数，甚至析构和构造函数，但是不能包含虚拟成员函数和静态数据成员，不能作为基类。

Enum

枚举类型允许我们定义特定用途的一组符号常量，它表明这种类型的变量可以取值的范围，如果不指定初始值，则第一个为0，后面依次递增1，如果指定了某一个，则它后面的依次在此基础上加1，它占用的字节数一般与int相同，但是不同编译器可能不一样，可以是short,byte等。
