IPC之Posix信号量

发表于 2016-06-14   |   分类于 IPC   |  

信号量的主要目的是提供进程间同步，互斥量和条件变量的意图是线程间同步，但是信号量也可以线程间同步，互斥量和条件变量也可以用于共享内存区进行进程间同步。

Posix信号量

POSIX信号量有两种：有名信号量和无名信号量，无名信号量也被称作基于内存的信号量。有名信号量通过IPC名字进行进程间的同步，而无名信号量如果不是放在进程间的共享内存区中，是不能用来进行进程间同步的，只能用来进行线程同步。

Posix信号量函数

有名信号量

sem_open

sem_open用于创建或打开一个信号量，信号量是通过name参数即信号量的名字来进行标识的，oflag参数可以为：0，O_CREAT，O_EXCL；mode参数用于创建信号量时，表示信号量的权限位；value是创建信号量的初值。
信号量有二值信号量（和互斥量相似）和计数信号量（多资源可用）。

IPC之Posix消息队列

发表于 2016-06-11   |   分类于 IPC   |  

Posix消息队列是Linux IPC的一种通信方式，通常用于在不同进程之间发送特定格式的消息数据。有足够写权限的线程可以往队列中放置消息，有足够读权限的线程可以从队列中读取消息。而且消息队列具有随内核的持续性，即使进程关闭了消息队列，只要引用计数大于0，此消息队列依然存在。

Posix消息队列基本函数

mq_open

创建一个消息队列，name为消息队列的名字，oflag和open函数类似，选项可以为O_RDONLY,O_WRONLY,O_RDWR,O_NONBLOCK,O_CREAT,O_EXCL等，mode是可选的，只有oflag中有O_CREAT并且消息队列不存在时候才需要，attr是属性设置，只能设置最大消息数目和消息的最大的size。返回消息队列的描述符。

mq_close

关闭一个打开的消息队列，但是并不从系统中删除，一个进程结束会自动关闭打开的消息队列。

STL之map按照key以及value排序

发表于 2016-06-07   |  

今天刷leetcode时候遇到题目Top K Frequent Elements，第一想法是使用map来记录每个元素的出现的次数，然后按照次数进行排序，这样取前k个即可，但是这就遇到一个问题，由于map默认是按照key排序的，呢么如何按照value排序呢，查了半天，终于找到解决之法，所以记录此文以备后用。首先介绍了map的按照key排序以及自定义排序准则的方法，然后介绍按照value排序的技巧。

按照key排序

map是默认按照key进行排序的，所以如果你这个key是自定义数据类型要规定排序准则，默认的为小于:

它有四个参数，其中我们比较熟悉的有两个: Key 和 Value。第四个是 Allocator，用来定义存储分配模型的。第三个参数： class Compare = less
这也是一个class类型的，而且提供了默认值 less。
less是stl里面的一个函数对象，那么什么是函数对象呢？所谓的函数对象：即调用操作符的类，其对象常称为函数对象（function object），它们是行为类似函数的对象。表现出一个函数的特征，就是通过“对象名+(参数列表)”的方式使用一个 类，其实质是对operator()操作符的重载。

当我们想要自己指定排序准则时候，只需要写一个函数对象,定义map的时候把Compare指定为我们自己编写的这个就ok啦。

miniftp项目实战之空闲断开，断点续传实现

发表于 2016-06-04   |   分类于 miniftp项目实战   |  

空闲断开分为控制连接的空闲断开和数据连接通道的空闲断开两方面。

控制连接空闲断开

空闲断开是说一段时间内没有任何命令，可以关闭连接，可以通过SIGALARM安装信号，并启动闹钟，在闹钟到来之前如果没有任何命令，在信号处理函数中关闭控制连接并给客户421 Timeout的响应，并退出会话。具体的是服务器端服务进程阻塞在readline中，在前面加上超时的alarm设置即可。

数据连接通道空闲断开

但是有一种情况，在数据传输过程中，服务进程不应该超时关闭，只需要将之前的闹钟关闭即可（重新安装闹钟即可）。另外数据传输之中有一定时间没有传输数据，则关闭。数据传输之前安装信号，并启动闹钟，数据传输过程中，收到SIGALARM信号，如果在传输过程中就设置传输标志为不传输，然后重新安装信号，否则回应Timeout响应并终止，当然这个时间在配置文件中有提到过，在数据传输前先设置超时时间，如果没有，也要设定闹钟为0，即关闭控制连接中的闹钟。
何时设置传输标志位，在限速函数中设定。

命令

RETR是进行文件下载的命令，在RETR后面如果有REST，则可以决定是否为断点续载;STOR是文件上传的命令，在STOR命令后面如果有REST，则可以决定是否为断点续传，另外APPE方式也可以进行文件的断点续传;

上传下载

上传下载命令是ftp的一个重大特性，那么如何实现呢？实际很简单，因为要有数据传输，所以要先建立数据连接通道，你可以PORT也可以PASV方式，然后就是数据传输，和列表传输大同小异，只不过列表传输的是列表的内容，而上传下载的是文件的内容，不过要主要文件进行加锁操作。
